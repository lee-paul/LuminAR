Testing Logs

Unit Tests were performed to ensure the functionality of the Unity plugins. Unit Tests were written before the code was produced, as per the test-first strategy defined in XP development methodology.

A Functionality Testing Log was produced at the end of each sprint. The log recorded the outcome of a series of functionality tests. These tests checked the functionality of the application against the acceptance tests of the User Stories.

Unit Tests

The Unit Tests were run before any code was released to clients, and to each other. This was to ensure that any changes in code had not affected functions that had already been implemented. It also ensured that the code was bug-free and working for showcase. A specific log for Unit Tests was not kept, it was agreed upon by the team to run the Unit Tests before any code was released.
